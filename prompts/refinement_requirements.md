# 本轮处理强制步骤

1. **理解全局**：先通读所有「当前轮次合并对话」，建立对本次轮次的整体叙事理解

2. **检查当前精炼状态**：阅读「当前精炼状态」了解已有的精炼结果。如果有已有精炼句子，需要结合本轮对话判断是否需要修改或删除

3. **ASR 纠错优先**：对合并对话中的明显同音字、错别字、断句错误进行最小必要纠正

4. **识别关键信息**：从每段合并对话中提取：
   - 剧情推进（地点变更、事件发生）
   - 决策/行动（队伍决定做什么）
   - 线索发现（证据、信息、观察）
   - 人物交互（NPC 对话、关系变化）
   - DM 的关键描述（场景、战斗、规则判定）

5. **精炼文本**：
   - 删除填充词、口头禅、重复内容
   - 合并冗余表达，保持一句话一个核心意图
   - 保留时间线和叙事顺序
   - 不编造原文中不存在的事实

6. **增量维护**：
   - 如果本轮对话与已有精炼句子冲突，优先 `refine edit` 修正旧句子
   - 如果已有精炼句子因新信息而不再准确，使用 `refine edit` 或 `refine remove`
   - 仅 `refine add` 真正的新增信息
   - 只有在本轮没有任何变化时才输出 `EMPTY`

7. **内容完整性检查**：确认没有遗漏对话中的关键信息
