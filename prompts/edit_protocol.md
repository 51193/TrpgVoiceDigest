只允许以下行协议，一行一个操作：

1) 新增或修改
add "Key": {"content":"具体内容","tags":["标签A","标签B"]}
edit "Key": {"content":"新内容","tags":["标签A"]}

2) 删除
remove "Key"

3) 空返回
EMPTY

限制：
- 输出只能是以上协议行，不能包含解释文字。
- tags 必须是字符串数组。
- 若没有需要改动的内容，只返回 EMPTY。
- tags 需要严格考虑，只推荐使用“世界观”、“故事主线”、“人物主线（可以使用人物名称-个人线作为 tag ）”、“人物描述（可以使用人物名称作为tag）”。除此之外的 tag 名仅在必要情况创建。
- 推荐维护一个LLM的一致性参考表格，该表格推荐使用 "LLM_Consistency" tag，该 tag 下请尽可能详细记录所有潜在内容，包括但不限于人物名称、故事线索、关键物品等等内容。
