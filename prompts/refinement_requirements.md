# 本轮处理强制步骤

1. **理解全局**：先通读所有「当前轮次对话」，建立对本次轮次的整体叙事理解

2. **检查当前精炼状态**：阅读「当前精炼状态」了解已有的精炼结果。如果已有精炼句子，需要结合本轮对话判断是否需要修改或删除

3. **ASR 纠错优先**：对对话中的明显同音字、错别字、断句错误进行最小必要纠正

4. **分类判断**：对每条对话内容逐一判断类型：
    - OOC（游戏外战术讨论、规则澄清、投骰讨论、闲聊）→ 直接丢弃
    - DM 描述环境/场景/氛围 → 记为 `[场景]`（这是重要优化，提升剧本感）
    - DM 间接转述 NPC 发言（"老板说…""他说道…"）→ **提取 NPC 身份或名称作为前缀**，如 酒馆老板、旅店老板
    - DM 直接扮演有明确身份的 NPC → 记为具体 NPC 名
    - DM 只在确实无法归类为以上情况时才保留 `DM：` 前缀（如纯规则说明、开篇泛泛介绍）
    - 角色扮演 → 记为角色名

5. **精炼文本**：
    - 删除填充词、口头禅、重复内容
    - 合并同一角色连续发言
    - 保留时间线和叙事顺序
    - 确保每一条结果都使用 `角色名：` 前缀，或 `[场景]：` 前缀
    - 积极将 DM 的叙述性发言拆分为 `[场景]` 描述和具体 NPC 对话
    - 对话中的角色名直接照搬使用

6. **增量维护**：
    - 仅对真正新增的信息使用 `refine add`
    - 如果本轮对话揭示了新的信息需要修改历史精炼（如角色自我介绍更正了之前的身份），使用 `refine edit` 或 `refine remove`
    - 只有在本轮没有任何变化时才输出 `EMPTY`

7. **输出检查**：确认结果中：
    - 每条都有正确的 `角色名：` 前缀
    - 没有 OOC 内容
    - 没有遗漏关键剧情信息
