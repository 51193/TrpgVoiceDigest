# 输出协议

你只能输出协议行或 `EMPTY`。**禁止**输出任何解释、Markdown、JSON 包裹文本或其他非协议内容。

## 协议格式

| 操作 | 格式 | 说明 |
|:---|:---|:---|
| 追加 | `refine add "文本"` | 在精炼结果末尾新增一条句子 |
| 插入 | `refine add 编号 "文本"` | 在指定编号的句子**之后**插入一条新句子，后续句子编号自动重排 |
| 编辑 | `refine edit 编号 "文本"` | 替换指定编号的句子内容 |
| 删除 | `refine remove 编号` | 删除指定编号的句子，后续句子自动重排编号 |
| 无变更 | `EMPTY` | 本轮无需任何变更 |

## 规则

1. 编号必须是整数，从 1 开始连续
2. 新产生的句子或批量修改的新内容，不能再次以编号形式重新引用
3. 每一行一个操作
4. 如果有多个独立的新增信息，必须分行编号输出多条 `refine add`
5. 如果两条操作内容相邻，但编号指向不同句子，必须分行（不允许合并成一次操作）
6. 若本轮操作覆盖同一编号多次（如先 edit 又 remove 同一个编号），必须按实际发生的顺序逐行输出

## 示例

```
refine add "队伍抵达酒馆门口，决定进入调查"
refine add 2 "酒馆大厅空无一人，但柜台上有新鲜血迹"
refine edit 1 "队伍经长途跋涉后抵达风暴港的酒馆门口"
refine remove 3
EMPTY
```

## 禁止的内容示例

```json
// 禁止：JSON 包装
{"operations": [{"action":"add","text":"..."}]}
// 禁止：Markdown
- refine add "..."
// 禁止：解释文字
我认为应该添加以下内容：refine add "..."
```
