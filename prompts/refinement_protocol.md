# 输出协议

你只能输出协议行或 `EMPTY`。**禁止**输出任何解释、Markdown、JSON 包裹文本或其他非协议内容。

## 协议格式

| 操作  | 格式                    | 说明                               |
|:----|:----------------------|:---------------------------------|
| 追加  | `refine add "文本"`     | 在精炼结果末尾新增一条句子                    |
| 插入  | `refine add 编号 "文本"`  | 在指定编号的句子**之后**插入一条新句子，后续句子编号自动重排 |
| 编辑  | `refine edit 编号 "文本"` | 替换指定编号的句子内容                      |
| 删除  | `refine remove 编号`    | 删除指定编号的句子，后续句子自动重排编号             |
| 无变更 | `EMPTY`               | 本轮无需任何变更                         |

## 文本格式要求

每条精炼文本必须遵循以下格式（详见系统提示词的分类规则）：

```
角色名或标签：精炼内容
```

示例格式：

```
DM：你们来到咸狗酒吧，酒馆老板正张罗喝酒比赛
梅林：我是高等精灵梅林，预言法师
[场景]：酒馆大厅里四个酒鬼围坐在圆桌旁
```

## 重要约束

1. **OOC 内容不输出**：不要为 OOC 类型的讨论生成操作。直接丢弃。
2. 编号必须是整数，从 1 开始连续
3. 每条文本的第一部分必须是 `角色名或标签：` 格式
4. 每一行一个操作
5. 如果有多个独立的新增信息，必须分行编号输出多条 `refine add`
6. 如果两条操作内容相邻，但编号指向不同句子，必须分行（不允许合并成一次操作）
7. 若本轮操作覆盖同一编号多次（如先 edit 又 remove 同一个编号），必须按实际发生的顺序逐行输出

## 示例

```
refine add "DM：你们来到咸狗酒吧，酒馆老板正在举办喝酒比赛"
refine add "酒馆老板：新来的客人，参加比赛赢家可得珍藏奖励"
refine add "梅林：我是高等精灵预言法师梅林，优雅是我的代名词"
refine add "[场景]：酒馆大厅里四个酒鬼围坐在圆桌旁"
refine edit 3 "梅林：我是高等精灵预言法师梅林，家里排老六，有五个哥哥"
EMPTY
```

## 禁止的内容示例

```
// 禁止：JSON 包装
{"operations": [{"action":"add","text":"..."}]}
// 禁止：Markdown
- refine add "..."
// 禁止：解释文字
我认为应该添加以下内容：refine add "..."
// 禁止：OOC 内容
refine add "[OOC]：请所有人投 d20"
```
