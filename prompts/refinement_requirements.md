# 本轮处理强制步骤

1. **理解全局**：先通读所有「当前轮次合并对话」，建立对本次轮次的整体叙事理解

2. **检查当前精炼状态**：阅读「当前精炼状态」了解已有的精炼结果。如果有已有精炼句子，需要结合本轮对话判断是否需要修改或删除

3. **ASR 纠错优先**：对合并对话中的明显同音字、错别字、断句错误进行最小必要纠正（例如 "龙与地下村" → "龙与地下城"，"冰封火" → "冰峰谷"）

4. **分类判断（关键步骤）**：对每条对话内容逐一判断类型：
   - `OOC`（游戏外）：战术讨论、规则澄清、投骰讨论、闲聊 → **直接丢弃，不写入精炼**
   - `场景`：主持人描述环境、氛围、布景 → 记为 `[场景]` 前缀
   - `DM`/`NPC名`：主持人叙述 NPC 言行、剧情推进 → 记为 `DM：` 或 `NPC名：`
   - `角色名`：玩家或 NPC 的角色扮演对话和行动 → 记为 `角色名：`
   
   输出格式：**角色名：精炼内容**（每条一行）

5. **识别关键信息**：从每条非 OOC 对话中提取：
   - 剧情推进（地点变更、事件发生）
   - 角色自述（人物名称、种族、职业、背景）
   - 决策/行动（角色决定做什么）
   - NPC 交互（与 NPC 的对话、关系变化）
   - DM 的关键描述（场景、剧情推进）

6. **精炼文本**：
   - 删除填充词、口头禅、重复内容
   - 合并同一角色连续发言
   - 保留时间线和叙事顺序
   - **OOC 内容直接跳过，不通过 refine add 写入**

7. **增量维护**：
   - 如果本轮对话与已有精炼句子冲突，优先 `refine edit` 修正旧句子
   - 如果已有精炼句子因新信息而不再准确，使用 `refine edit` 或 `refine remove`
   - 仅 `refine add` 真正的新增信息

8. **输出检查**：确认结果中：
   - 每条都有 `角色名：` 前缀
   - 没有 `[OOC]` 类型的句子（已在步骤 4 排除）
   - 没有遗漏关键剧情信息
