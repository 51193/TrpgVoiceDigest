# 本轮处理步骤

1. **通读精炼结果**：先通读所有精炼文本，建立对当前故事进展的整体理解。

2. **检查当前状态**：阅读"当前故事进展状态"中的已有条目。判断已有事件是否需要修改、合并或删除。

3. **提取新章节**：从精炼文本中识别以下类型的**章节级叙事段落**：
    - **场景转换**：玩家从 A 地点移动到 B 地点，新地点的描述和抵达过程
    - **重要遭遇**：与重要 NPC 的完整互动过程（从初见 → 对话 → 结果）
    - **关键决策**：玩家做出的影响故事走向的重大选择及其上下文
    - **战斗/冲突**：完整的战斗或冲突弧（起因 → 过程 → 结果）
    - **信息揭示**：玩家获取到关键情报、线索或世界观信息的完整过程
    - **状态变化**：角色获得的物品、能力、关系变化等有长期影响的改变

4. **合并微观事件**：一个完整的叙事段落可能包含多个小事件（如 NPC 说话、检定、动作细节等），应将它们在一条进展中合并叙述，而非拆成多条。

5. **撰写叙事摘要**：每条进展应是一段连贯的叙事段落，包含：
    - **地点/场景**：事件发生的位置和环境
    - **人物行动**：谁做了什么，为什么
    - **关键对话要点**：传达了什么重要信息
    - **结果/影响**：事件对后续故事的影响

6. **增量维护**：
    - 新事件用 `story add` 追加
    - 遗漏的早期事件用 `story add <编号>` 插入到合适位置
    - 随着理解加深，旧条目可能需要合并或改写，用 `story edit` 修正
    - 过时或过于细碎应被合并的旧条目用 `story remove` 删除
    - **窗口化注意**：你收到的状态列表是窗口化切片，部分条目被省略（普遍是前文被省略）。

7. **提取活跃任务**：从精炼文本中识别当前活跃的任务：
    - **主线任务**：DM 或 NPC 明确提出的、推动主线剧情的目标
    - **支线任务**：玩家主动提出的目标、NPC 的委托
    - 任务描述应简洁明确，说明目标和背景
    - 新任务用 `task add` 追加
    - 任务完成用 `task complete <编号>` 标记成功
    - 任务失败或放弃用 `task fail <编号>` 标记失败
    - 任务描述变化用 `task edit <编号>` 修正
    - 重复或错误的活跃任务用 `task remove <编号>` 删除

8. **输出检查**：确保所有输出行符合对应的协议格式，无额外内容。
