# 本轮处理强制步骤

1. **理解全局**：先通读所有「当前轮次合并对话」，建立对本次轮次的整体叙事理解

2. **检查当前精炼状态**：阅读「当前精炼状态」了解已有的精炼结果。如果已有精炼句子，需要结合本轮对话判断是否需要修改或删除

3. **解析说话人映射表**（关键）：
   - 查看「说话人映射表」，了解哪些 speaker 已识别、哪些未识别
   - 对于「未识别」的 speaker，在通读对话时同步推断其真实身份
   - 判定规则：谁进行自我介绍 → 使用名字；谁在叙述场景/规则 → DM；谁被称为某名 → 使用该名

4. **ASR 纠错优先**：对合并对话中的明显同音字、错别字、断句错误进行最小必要纠正

5. **分类判断**：对每条对话内容逐一判断类型：
   - OOC（游戏外战术讨论、规则澄清、投骰讨论、闲聊）→ 直接丢弃
   - 场景描述 → 记为 `[场景]`
   - DM 叙述/裁决 → 记为 `DM`
   - DM 扮演 NPC → 记为具体 NPC 名
   - 角色扮演 → 记为角色名

6. **精炼文本**：
   - 删除填充词、口头禅、重复内容
   - 合并同一角色连续发言
   - 保留时间线和叙事顺序
   - 确保每一条结果都使用 `角色名：` 前缀（**禁止出现 speaker_X**）

7. **修正已有精炼中的 speaker_X**（最高优先级）：
   - 遍历「当前精炼状态」中所有已有句子
   - 如果发现任何句子包含 `speaker_X`，必须使用 `refine edit` 将其替换为正确的角色名或 DM
   - 此操作优先于 `refine add`（先修旧，再加新）

8. **增量维护**：
   - 仅对真正新增的信息使用 `refine add`
   - 如果本轮对话揭示了某个 speaker 的真实身份，修正所有受影响的历史句子
   - 只有在本轮没有任何变化时才输出 `EMPTY`

9. **输出检查**：确认结果中：
   - 没有任何 `speaker_X` 残留（无论是新增还是已有句子）
   - 每条都有正确的 `角色名：` 前缀
   - 没有 OOC 内容
   - 没有遗漏关键剧情信息
