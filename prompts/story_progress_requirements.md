# 本轮处理步骤

1. **通读精炼结果**：先通读所有精炼文本，建立对当前故事进展的整体理解。

2. **检查当前状态**：阅读"当前故事进展状态"中的已有条目。判断已有事件是否需要修改、合并或删除。

3. **提取新事件**：从精炼文本中识别以下类型的事件：
   - **场景转换**：玩家从 A 地点移动到 B 地点（尤其是第一次到达新地点）
   - **重要信息获取**：玩家获取到关键情报、线索或背景信息
   - **角色决策**：玩家做出的影响故事走向的选择
   - **NPC 互动**：与重要 NPC 的有意义互动（加入、离开、冲突、结盟等）
   - **战斗/冲突**：发生重要战斗或冲突事件
   - **状态变化**：角色的重要状态变化（获得物品、受伤、获得能力等）

4. **增量维护**：
   - 新事件用 `story add` 追加
   - 遗漏的早期事件用 `story add <编号>` 插入到合适位置
   - 不再准确的事件用 `story edit` 修正
   - 过时的事件用 `story remove` 删除

5. **提取活跃任务**：从精炼文本中识别当前活跃的任务：
   - **主线任务**：DM 或 NPC 明确提出的、推动主线剧情的目标
   - **支线任务**：玩家主动提出的目标、NPC 的委托
   - 任务描述应简洁明确，说明目标和背景
   - 新任务用 `task add` 追加
   - 任务完成用 `task complete <编号>` 标记成功
   - 任务失败或放弃用 `task fail <编号>` 标记失败
   - 任务描述变化用 `task edit <编号>` 修正
   - 重复或错误的活跃任务用 `task remove <编号>` 删除

6. **输出检查**：确保所有输出行符合对应的协议格式，无额外内容。
